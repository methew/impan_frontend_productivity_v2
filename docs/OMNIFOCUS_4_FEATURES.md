## 核心概念

### 基础实体

| 实体 | 说明 | 图标 |
|------|------|------|
| **Inbox Item** | 收件箱项目 - 快速捕获的想法 | 📥 |
| **Action** | 行动 - 可执行的最小单位 | ⭕ |
| **Action Group** | 行动组 - 包含子任务的任务 | 📁 |
| **Project** | 项目 - 有目标的行动集合 | 📋 |
| **Folder** | 文件夹 - 组织项目的容器 | 📂 |
| **Tag** | 标签 - 分类标记 | 🏷️ |

### 项目类型

```
1. Sequential (顺序型)
   - 任务必须按顺序完成
   - 只有第一个可用任务是活跃的

2. Parallel (并发型)
   - 任务可以任意顺序完成
   - 所有任务同时可用

3. Single Actions (单个行动)
   - 不相关的独立任务集合
   - 没有完成目标
```

## 标准透视 (Perspectives)

### 1. Inbox (收件箱)

**用途**: 快速捕获和组织任务

**功能**:
- 快速添加新任务 (Quick Entry)
- 批量处理收件箱项目
- 分配项目、标签、日期
- 清理收件箱 (Process Inbox)

**界面元素**:
- 任务计数
- 项目分配按钮
- 标签选择器
- 状态圆圈

**快捷键**:
- `Cmd + N` - 新建任务
- `Cmd + Shift + N` - 快速录入

---

### 2. Projects (项目)

**用途**: 组织和管理项目

**功能**:
- 查看所有项目和文件夹树
- 创建/编辑/删除项目
- 设置项目类型 (Sequential/Parallel/Single Actions)
- 查看完成百分比
- 添加项目备注

**界面布局**:
```
+------------------------------------------+
|  Sidebar  |     Outline (Project Tree)   |
|  (Folders)|                              |
|           |  📁 Folder A                 |
|           |    📋 Project 1    [60%]     |
|           |    📋 Project 2    [30%]     |
|           |  📁 Folder B                 |
|           |    📋 Project 3    [100%] ✅ |
+------------------------------------------+
```

**项目状态**:
- 🟢 Active - 活跃
- ⏸️ On Hold - 暂停
- ✅ Completed - 已完成
- 🗑️ Dropped - 已丢弃

### 3. Tags (标签)

**用途**: 多维度分类任务

**标签类型**:

- **Energy Level** - 能量等级 (High/Medium/Low)
- **Priority** - 优先级
- **Context** - 上下文标签

```
└── ⚡ Energy
    ├── 🔥 High

    └── 🍃 Low
```

**Nearby 功能**:

**用途**: 定期回顾项目状态

## 任务状态系统

### Status Circle (状态圆圈)

#### 视觉状态

| 状态 | 外观 | 说明 |
|------|------|------|
| **Active** | ⭕ 空心圆圈 | 可操作的任务 |
| **Completed** | ✅ 绿色填充+对勾 | 已完成 |
| **Dropped** | ➖ 灰色填充+横线 | 已丢弃 |
| **Due Soon** | 🟡 黄色圆圈 | 即将截止 |
| **Overdue** | 🔴 红色圆圈 | 已逾期 |
| **Flagged** | 🟠 橙色圆圈+旗帜 | 已标记 |
| **Repeating** | ⭕ 圆圈+三点 | 重复任务 |

#### 状态组合示例

```
🔴➕🚩➕⋯ = 逾期 + 已标记 + 重复
(红色圆圈，橙色旗帜，中间三点)
```

#### 交互

- **单击** - 标记完成
- `Option` + **单击** - 标记丢弃
- **空格键** - 切换完成状态
- `Option` + **空格键** - 切换丢弃状态

## 日期系统

### 三种日期类型

```
时间轴:
─────────────────────────────────────────►

Defer Date          Planned Date      Due Date
   │                    │               │
   ▼                    ▼               ▼
  可用               计划执行         必须完成
  (Unavailable)       (Scheduled)    (Deadline)
   │                    │               │
   └────────────────────┴───────────────┘
           任务生命周期
```

### 1. Defer Date (推迟日期)

**用途**: 任务何时变为可用

**行为**:
- 在推迟日期前，任务处于 **Unavailable** 状态
- 不会出现在大部分透视中
- 用于隐藏未来的工作

**继承规则**:
- 父项目/任务组的推迟日期影响所有子任务
- "Start No Earlier Than" 约束

### 2. Due Date (截止日期)

**用途**: 任务必须完成的时间

**行为**:
- 接近时变为 **Due Soon** (黄色)
- 过期后变为 **Overdue** (红色)
- 影响状态圆圈颜色

**继承规则**:
- 父项目/任务组的截止日期影响所有子任务
- "End No Later Than" 约束

### 3. Planned Date (计划日期) ⭐ 4.7+

**用途**: 你实际打算执行的时间

**与 Defer/Due 的区别**:
| 特性 | Defer | Due | Planned |
|------|-------|-----|---------|
| 影响任务状态 | ✅ 是 | ✅ 是 | ❌ 否 |
| 用于日程规划 | ❌ 否 | ⚠️ 部分 | ✅ 是 |
| 灵活性 | 低 | 低 | 高 |
| 发送通知 | ✅ 可用时 | ✅ 截止时 | ✅ 计划时 |

**使用场景**:
- 你想在周三做，但周五才截止
- 你想明天做，但下周才可用
- 需要灵活的日程安排

**继承规则**:
- 仅作为子任务的默认值
- 不创建硬性约束
- 可以单独调整子任务

## 界面布局

### Fluid Layout (流式布局)

**默认字段**:
```
+--------------------------------------------------+
| ⭕ 任务标题                                       |
|    📁 项目名称  🏷️ 标签1 标签2  📅 截止日期      |
```

**编辑时显示**:
- 备注 (Notes)
- 推迟日期
- 计划日期
- 重复设置
- 位置
- 附件

### Columns Layout (列布局) - Mac Only

**列配置**:
```
+--------------------------------------------------+
| ⭕ | 任务 | 项目 | 标签 | 截止日期 | 标记 | 重复 |
+--------------------------------------------------+
```

### Sidebar (侧边栏)

**内容**:
- 透视列表
- 文件夹树
- 标签树
- 过滤器

**显示模式**:
- 常驻显示 (Mac/iPad)
- 滑入滑出 (iPhone)

### Inspector (检查器)

**标签页**:
1. **Overview** - 概览信息
2. **Dates** - 日期设置
3. **Repeat** - 重复规则
4. **Tags** - 标签分配
5. **Notes** - 备注和附件

## 快捷功能

### Quick Entry (快速录入)

**全局快捷键**:
- `Ctrl + Option + Space` - 打开快速录入

**功能**:
- 从任何应用快速添加任务
- 自动填充当前应用信息
- 支持富文本备注

### Quick Open (快速打开)

**快捷键**:
- `Cmd + O` - 打开快速导航

**功能**:
- 搜索透视、文件夹、项目、标签
- 支持模糊搜索
- 键盘导航 (上下箭头 + 回车)
- 打开新窗口 (Mac)

### Batch Editing (批量编辑)

**操作**:
1. 选择多个任务 (`Cmd` + 点击)
2. Inspector 显示批量编辑界面
3. 可同时设置:
   - 标签
   - 截止日期
   - 推迟日期
   - 计划日期
   - 项目

## Pro 功能

### 1. Custom Perspectives (自定义透视)

**创建方式**:
```
透视编辑器
├── 过滤规则 (Filter Rules)
│   ├── 状态: Active/On Hold/Completed/Dropped
│   ├── 可用性: Available/Unavailable
│   ├── 日期: Due/Deferred/Planned
│   ├── 标签: 包含/不包含
│   └── 项目: 属于/不属于
├── 分组方式 (Group By)
│   ├── 项目
│   ├── 标签
│   ├── 截止日期
│   ├── 推迟日期
│   └── 文件夹
└── 排序方式 (Sort By)
    ├── 截止日期
    ├── 创建日期
    ├── 优先级
    └── 字母顺序
```

**示例自定义透视**:
- "High Energy Work" - 高能量 + 工作标签
- "This Week" - 本周截止 + 活跃状态
- "Waiting For" - 等待标签 + 按人员分组

### 2. Focus Mode (专注模式)

**用途**: 临时隐藏不相关的任务

**使用方式**:
1. 选择要专注的项目/文件夹/标签
2. 点击 Focus 按钮
3. 只显示相关内容
4. 其他任务被隐藏

**应用场景**:
- 工作时间专注工作项目
- 在家隐藏工作相关内容
- 专注特定客户的项目

### 3. Apple Shortcuts 集成

**支持操作**:
- 添加任务
- 查找任务
- 完成任务
- 获取透视

### 4. Omni Automation (JavaScript)

**用途**: 自动化和插件开发

**示例脚本**:
```javascript
// 批量添加标签
var tasks = document.windows[0].selection.tasks
 tasks.forEach(task => {
   task.addTag(Tag.named("Today"))
})
```
